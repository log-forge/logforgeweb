Built for Fast-Moving Teams Using Docker

LogForge is the developer-first monitoring tool made for Dockerized workflows. It helps fast-moving teams and startups stay confident in production-like dev environments, and iterate quickly.

Speed up dev cycles, fix faster. Ship faster.

Install LogForge Now

Self-host in seconds with Docker. One command to get started: 

git clone https://github.com/log-forge/logforge.git
cd logforge
docker compose up -d --build
Copied!

Monitor your containers. Tail logs. Get alerts. All in one lightweight dashboard built for devs.

🛠 Service Overview

Autodetect and list services from Docker. See real-time status and control your services from the UI.

📜 Logs, Terminals & Files

Filter for errors, warnings, or custom keywords. Access an interactive terminal and browse container files directly from your browser.

🚨 Smart Alerts

Subscribe to containers. Get instant email alerts for errors that matter — per-container, per-keyword.

⚡ One Command Setup

No agents, no config hell. Just run docker compose up -d --build and go.

📈 Built for Speed

Minimal overhead. Zero config. Get started in seconds, scale as needed — no learning curve.

📊 Not Just Another Log Viewer

Always-on, local, and alert-driven — built to speed up dev cycles and help developers focus on what actually matters.

🤖Cursor for Your Docker Container

AI agents that live inside each container — reading logs, explaining crashes, and helping you fix or improve your code with context-aware suggestions.
